# 코딩과 디버깅에 관하여.

- `이상적인 세계`에서는 한 함수가 두 가지 이상의 일을 해서는 안된다고 말함.
 1. 입력을 읽어들이는 함수.
 2. 입력을 처리하기 쉬운 형태로 바꾸는 함수
 3. 실제 문제를 푸는 함수

- 같은 자료를 두 가지 형태로 저장하지 않는 것.
 - `이상적으로`는 자료를 표현하는 클래스의 생성자에서 정규화를 수행하거나, 외부에서 자료를 입력받자마자 정규화를 수행하는 것이 좋음.


- 코드와 데이터를 분리

- Off-by-one 오류
 - 계산의 큰 줄기는 맞지만, 하나가 모자라거나 하나가 많아서 틀리는 코드의 오류들을 통칭.

- 잘못된 비교 함수 작성.
 - `C++`는 `<` 연산을 비교 함수의 모델로 사용.
 - `Java`는 `<=` 연산을 비교 함수의 모델로 사용.
 - `.NET`은 `bool`값 대신, `양수`, `0`, `음수`를 반환함.

- 자신이 사용하는 언어에서 입출력방식이 어떤것들을 지원하는지, 빠른지를 미리 확인하자.

- 코드가 복잡해지면, 디버거 없이 눈으로 프로그램을 검증하기 어려움
 - 애초에 코드를 복잡하게 짰다는 것이 문제.
 - 잘 분리된 기능적인 코드는, 디버거 없이 눈만으로도 검증하기가 비교적 쉬움.

- `987,654,321`
 - 2^30에 가까운 값이면서, 32비트 부호있는 정수에 오버플로 내지않고, 오타가 났는지 눈으로 확인 용의.

- IEEE 754
